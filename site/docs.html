Informations
Introduction
The Discord ZT API was created on August 10, 2022 (the first version was usable on 08/20/2022).
The purpose of this API is to allow third-party applications to easily retrieve information from the Zone-telechargement site without having to do scraping.


Why this API ?
The purpose of this API is to allow third-party applications to easily retrieve information from the Zone-telechargement site without having to do scraping.

How to use it ?
Its really simple. With your API token you make a request to our API asking what do you want, its entierely customizable..

Get early access
Unfortunatly the API is still in a pre-beta version, meaning that its not public.
But keep in mind that if you're using this API in pre-beta / beta / alpha / canary and any other unstable versions, you'll have to often check our documention to see if there is no modifications.
You can find the versions changelogs in the left-side menu.

Settings Page
Settings Structure
Endpoint to create the setting pages.
POST /api/createPageSetting

Body Params
FIELD
TYPE
DESCRIPTION
pageName
string
The name of the setting page (your application name in general)
?pageDescription
string
The name of the setting page (your application name in general)
backToEndpointToken*
string
The Token of the endpoint to send back the response
backToEndpointUrl
string
The URL to send back the response (PUT method, see reponse DBS API request format)
settingsWaterfall
array
The waterfall of settings. see waterfall settings list.
?oneUse*
boolean
If the page will expires once it is used.
?backBody
object
The body which will be sent back to your application without being modified.
?param means that the parameter is optional
* In the version 1.0.0, this param does not yet have an utility, juste set the field the any string, it wont matter.
** In the version 1.0.0, this param is required to be true.
